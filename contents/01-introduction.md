# はじめに

このテキストは、Rustでドメイン駆動開発をどのように実現するかを、概念とコードの両面から整理するための学習用コンテンツです。

一般的なDDDの解説は、オブジェクト指向言語を前提に書かれていることが多く、Rustにそのまま持ち込むと設計上の違和感が生まれます。たとえば、可変なオブジェクトグラフを前提にした設計や、継承による責務分担は、Rustでは中心的なアプローチになりません。

一方で、RustにはDDDと相性のよい特徴があります。

- newtype pattern によって、ドメイン固有の意味を型として明示できる
- コンストラクタで妥当性検証を行い、不正な状態の生成を防ぎやすい
- enum によって状態遷移や失敗理由を明示的に表現できる
- 所有権と借用の制約が、曖昧な責務分散を減らす方向に働く

本書では、まずDDDの基本用語をRustの文脈で読み替え、その後で小さなタスク管理ドメインを題材に実装を進めます。サンプルコードは小さなCargoプロジェクトを前提にし、各章で紹介した設計をテストで検証できる形で示します。

## 本書の進め方

各章は次の流れを基本にします。

1. その章で扱う概念を整理する
2. Rustでどう表現するかを設計観点から説明する
3. サンプルコードに対応づけて確認する
4. 実務で迷いやすい点や設計上の注意点を補足する

## 最初に押さえたい前提

このテキストで重視するのは、DDDの用語を表面的になぞることではありません。重要なのは、ドメインのルールをコード上でどのように守るかです。

そのため、ここでは次の方針を取ります。

- フレームワーク中心ではなく、ドメインモデル中心で説明する
- 小さく検証可能なコードを先に作る
- 永続化やWeb APIの都合は、ドメインの都合から分離して考える

以降の章では、まず「なぜRustでDDDを学ぶのか」を整理し、その上で具体的な実装パターンに入ります。
