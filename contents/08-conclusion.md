# 第8章 まとめ

このテキストでは、RustでDDDを実践するときの最小構成を、概念とコードの両方から整理しました。

## 本書で扱ったこと

- Value Objectで意味と制約を型に閉じ込める
- Entityに振る舞いを集め、不変条件を守る
- Aggregate境界を小さく明確に取る
- Application Serviceでユースケースを調停する
- Repositoryで永続化境界を抽象化する
- インメモリ実装とテストで設計を検証する

## 重要だった観点

最も重要なのは、DDDの用語を並べることではなく、ドメインルールをどこに置くかを説明できることです。Rustは、その説明を型と関数の制約として書きやすい言語です。

逆に言えば、RustでDDDをやるときは、他言語の定番パターンをそのまま写経するよりも、Rustの型システム、所有権、enum、Result をどう活かすかを先に考えるべきです。

## 次の一歩

次に進むなら、次のいずれかが自然です。

1. 今のタスク管理ドメインに期限や担当者を追加する
2. SQLite などを使ったRepository実装を加える
3. Web APIやCLIから Application Service を呼び出す
4. 読み取りモデルやDomain Eventへ発展させる

学習用としては、まず小さなドメインで責務分担が保てているかを確認し、そのあとで拡張していくのが堅実です。
